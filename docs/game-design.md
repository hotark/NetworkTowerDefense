# Network Tower Defence - ゲーム仕様書

## 概要

ネットワークをテーマにしたタワーディフェンスゲーム。
プレイヤーは拠点を守るために、ノード（タワー）を設置しエッジで接続する。
パケットがエッジを通じてネットワーク上を流れ、攻撃や建設のリソースとなる。

---

## コアコンセプト

### ネットワーク構造

| 要素 | 説明 |
|------|------|
| **拠点（ベース）** | プレイヤーの防衛対象。パケットの起点。敵がここに到達するとダメージ |
| **ノード（タワー）** | プレイヤーが配置するタワー。種類によって異なる機能を持つ |
| **エッジ** | ノード・拠点間を結ぶ接続線。パケットの通り道 |
| **パケット** | エッジ上を流れるリソース。攻撃の弾薬かつ建設・強化の通貨 |

### 敵

- 種類: **1種類のみ**（基本の敵）
- 経路: マップ端のスポーン地点から固定ルートを通り、拠点へ向かう
- ルートには分岐があるが、基本的に拠点へ一方的に進む

---

## タワー（ノード）

### 1. パケット生成器

- **役割**: パケットを一定間隔で生成する
- **仕組み**: 生成したパケットは接続されたエッジに送り出される
- **備考**: ネットワークの供給源。これがないとパケットが流れない

### 2. 攻撃タワー

- **役割**: 敵を攻撃する
- **仕組み**: パケットを弾薬として消費し、範囲内の敵にダメージを与える
- **備考**: パケットが届かなければ攻撃できない

### 3. 分配器（ルーター）

- **役割**: パケットの経路を分岐・制御する
- **仕組み**: 受け取ったパケットを接続された複数のエッジに振り分ける
- **備考**: パケットの流れを効率的にコントロールするための中継ノード

---

## パケット

### 二重の役割

1. **弾薬**: 攻撃タワーに届くと消費され、敵への攻撃となる
2. **リソース**: 新しいタワーの建設やアップグレードに使用できる

### 流れ

```
パケット生成器 → エッジ → [分配器] → エッジ → 攻撃タワー（消費）
                                    → エッジ → 別のタワーへ
```

---

## 敵の経路

```
スポーン地点（マップ端）
    │
    ▼
  固定ルート（分岐あり）
    │
    ▼
  拠点（プレイヤーのベース）
```

- 道は事前に固定されている
- 分岐点では敵がいずれかのルートを選択する
- 敵が拠点に到達するとプレイヤーにダメージ

---

## ゲームフロー（暫定）

1. **準備フェーズ**: タワーの配置・エッジの接続
2. **ウェーブ開始**: 敵がスポーンし拠点へ向かう
3. **防衛**: パケットが流れ、攻撃タワーが敵を迎撃
4. **ウェーブ間**: 報酬でタワーの追加・強化
5. **繰り返し**: ウェーブが進むごとに敵が強化

---

## 技術（予定）

- **ゲームエンジン**: Unity
- **現段階**: モックアップ作成（プロトタイプ）
