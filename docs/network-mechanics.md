# ネットワークメカニクス設計書

## コンセプト

「タワーが強くなる」のではなく「ネットワークトポロジが強くなる」ゲーム。
実際のネットワーク用語（輻輳・冗長性・優先度・帯域）がそのままゲームメカニクスになる。

---

## 1. 輻輳（Congestion）

### 概要
エッジには帯域制限があり、パケットが集中するとボトルネックが発生する。

### ルール
- エッジごとに **帯域 (bandwidth)** を持つ（例: Lv1 = 3パケット/秒）
- 帯域を超えるとパケットが **遅延** する（速度低下）
- さらに超過すると **パケットロス** が発生（消失）
- プレイヤーは混雑を視覚的に確認できる（エッジの色変化、太さ変化）

### 戦略的意味
- 1本のエッジに頼ると詰まる → 分散設計を促す
- 後半ウェーブで敵が増える → 帯域不足が自然に発生 → トポロジ拡張が必要

### エッジアップグレード
| レベル | 帯域 | コスト |
|--------|------|--------|
| Lv1 | 3 pkt/s | 初期 |
| Lv2 | 5 pkt/s | 中 |
| Lv3 | 8 pkt/s | 高 |

## 2. 冗長性（Redundancy）

### 概要
同じタワーへの経路を複数用意することで、障害耐性を確保する。

### ルール
- 敵がエッジを攻撃して **切断** することがある
- エッジが切れても代替経路があれば弾薬供給が継続
- 冗長経路はコスト（エッジ建設費 + 中継ノード）がかかる

### 戦略的意味
- 安い1本道 vs 高い冗長構成 のリスク判断
- 後半は敵がエッジ破壊を狙ってくる → 冗長化が必須に
- 冗長構成を組めるプレイヤーが生き残る

### 障害パターン
- **エッジ切断**: 敵の特殊攻撃でエッジHP → 0
- **ノード破壊**: ノードが壊れると接続エッジも全滅
- **自動復旧**: 修理コストを払えば再接続可能

---

## 3. 優先度（QoS / Priority）

### 概要
パケットに優先度を設定し、重要な経路に優先的にリソースを回す。

### ルール
- 分配器で **優先度ルーティング** を設定可能
- 高優先度パケットは帯域制限時でも先に通過
- 低優先度は余剰帯域でのみ配送

### 優先度レベル（案）
| レベル | 用途例 |
|--------|--------|
| 高 | 前線の攻撃タワー（敵に近い） |
| 中 | 通常配送 |
| 低 | 後方の備蓄・修理用 |

### 戦略的意味
- 前線が危ない時に自動で弾薬を集中させる
- 余裕がある時は後方にもリソースを回す
- 手動設定 or 自動ルールの選択

## 4. スケーラビリティ（進行感）

### 核心
タワーを強化すると弾薬消費が増える → 今のネットワークでは供給が追いつかない → ネットワーク側の強化が必要になる。
タワー強化がネットワーク強化を**要求する**構造。

### ボトルネックの連鎖

```
タワーLv1 → 弾薬消費 1/秒 → エッジLv1で十分
    ↓ アップグレード
タワーLv2 → 弾薬消費 3/秒 → エッジLv1だと輻輳発生！
    ↓ エッジ強化 or リピーター追加
タワーLv3 → 弾薬消費 6/秒 → エッジ強化 + 冗長経路が必要
    ↓ 分配器でロードバランス
タワーLv4 → 弾薬消費 10/秒 → 複数生成器 + 高帯域ネットワーク必須
```

つまり「タワーを強くしたい → でもネットワークが足りない → 先にネットワークを整備する」
というサイクルが回る。

### ステージ進行イメージ

```
序盤（Wave 1-5）
  拠点 ──── タワーLv1
  1本道で十分。タワーの弾薬消費も少ない。

中盤（Wave 6-15）
  拠点 ──── リピーター ──── タワーLv2
       └──── 分配器 ──── タワーLv2
  タワーを強化したら帯域不足に。リピーターや分岐で供給増。

終盤（Wave 16-30）
  生成器 ═══ リピーター ═══ 分配器 ═══ タワーLv3
  拠点 ═══════════════════════┘   └═══ タワーLv3
       └══ 冗長経路 ═══ 分配器 ═══ タワーLv4
  (═ は強化済みエッジ)
  高レベルタワーを支えるための太い幹線網。
```

### 進行感の変化

| フェーズ | プレイヤーの課題 | 感覚 |
|----------|-----------------|------|
| 序盤 | タワーを置いて繋ぐ | 「繋げば動く」 |
| 中盤 | タワー強化したいけど帯域足りない | 「ネットワーク整備しなきゃ」 |
| 終盤 | 高火力タワーを支える大規模網 | 「インフラ構築してる」 |

### タワーレベルとネットワーク要求の対応（案）

| タワーLv | 弾薬消費 | 必要な供給インフラ |
|----------|----------|-------------------|
| Lv1 | 低 | エッジ1本 |
| Lv2 | 中 | エッジ強化 or リピーター |
| Lv3 | 高 | 強化エッジ + リピーター |
| Lv4 | 極高 | 複数経路 + 高帯域 + ロードバランス |

## 5. コスト設計の方針

### ネットワーク系は高コスト
分岐やリピーターはネットワークを質的に強化するため、コストを高めに設定。
タワー単体より、ネットワーク構築にリソースを使う設計。

| 要素 | コスト感 | 理由 |
|------|----------|------|
| 攻撃タワー | 低〜中 | 数が必要、ネットワークの末端 |
| パケット生成器 | 中 | 供給源、序盤から必要 |
| リピーター | **高** | パケット増幅、ネットワーク価値を倍化 |
| 分配器 | **高** | 分岐、トポロジの要 |
| エッジ | 低（本数で積む） | 1本は安いが冗長化するとコスト嵩む |
| エッジ強化 | 中〜高 | 帯域拡張、後半必須 |

## 6. ビジュアルとネットワーク感

### エッジの状態を可視化
- **帯域使用率**: 色や太さで表現（青→黄→赤で混雑度）
- **パケット密度**: エッジ上のパケットが視覚的に詰まる
- **障害**: ひび割れ、点滅で損傷を表示

### ネットワーク全体の「生きてる感」
- パケットが常に流れている → ネットワークが脈動している
- 輻輳エッジはパケットが渋滞して見える
- 冗長経路に切り替わる瞬間が分かる

### スケール感の演出
- 序盤: 数本のエッジと数ノード、静かな画面
- 終盤: 大量のエッジとノード、パケットが縦横無尽に走り回る
- 「自分が構築したネットワーク」が画面を埋め尽くす達成感

### タワーレベル別ビジュアル

タワーのレベルが上がるごとに形状・エフェクトが変化。
見た目で「このタワーは高レベル = 重いインフラが必要」と一目で分かる。

| Lv | 形状 | アイドルエフェクト | 発射エフェクト |
|----|------|-------------------|---------------|
| Lv1 | シンプルな四角 | なし | 小フラッシュ |
| Lv2 | 八角形、少し大きい | 微かな脈動グロー | フラッシュ + スパーク |
| Lv3 | 多角形 + 内部構造線 | 回転リング + グロー | リング波 + スパーク放射 |
| Lv4 | 複雑な形状 + オーラ | 浮遊パーティクル + 電弧 | 衝撃波 + 多重リング |

### アップグレード演出（重要）

レベルアップの瞬間は、このゲーム最大のカタルシスポイント。
タワー強化 = ネットワーク投資の成果が実る瞬間なので、派手に演出する。

- **演出の流れ**:
  1. エネルギー収束（周囲のパーティクルがタワーに集まる）
  2. フラッシュ（白い閃光）
  3. 形状変化（旧形状 → 新形状にモーフィング）
  4. リング波放出（完成を周囲に伝播）
  5. 新しいアイドルエフェクト開始

- **ネットワーク連動演出**:
  - アップグレード中、接続エッジにエネルギーが逆流する表現
  - 完了時、接続エッジが一瞬明るく光る（ネットワーク全体に波及）
  - 帯域不足の場合、警告表示（赤く点滅 →「エッジ強化が必要」のヒント）

### アップグレード中の停止とデュアルシステム

タワーはアップグレード中に**動作停止**する。
これにより「いつ・どれを上げるか」が戦略的判断になる。

- **停止リスク**: 前線タワーを強化中、その区間の防衛が空く
- **対策 = 冗長構成**: 同じ区間をカバーする2台目のタワーを用意
- **デュアルシステム**: A/Bタワーを交互にアップグレードすることで防衛を途切れさせない

```
パターン1: 無計画（危険）
  タワーA [アップグレード中...停止] ← 敵素通り！

パターン2: デュアルシステム（安全）
  タワーA [アップグレード中...停止]
  タワーB [稼働中→防衛カバー]    ← Aの分もBが守る
  → Aが完了したらBをアップグレード
```

- **ネットワーク的な含意**:
  - デュアル構成にはパケット供給も2系統必要 → ネットワーク投資が倍
  - でもそれをやらないとアップグレードできない → 自然にネットワーク拡張が促される
  - 上級プレイヤーは「メンテナンスウィンドウ」を計画的に作る

### リピーター・分配器のレベル別ビジュアル

ネットワーク系ノードもレベルで見た目が変わる。

| ノード | Lv1 | Lv2 | Lv3 |
|--------|-----|-----|-----|
| リピーター | 単色の増幅マーク | 脈動 + 周回パーティクル | エネルギーリング二重 |
| 分配器 | 分岐の矢印マーク | 回転する分岐アイコン | 光の分岐線が常時表示 |
